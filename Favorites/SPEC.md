## 1. 프로젝트 목표

최신 제약·의약품 뉴스를 **한 눈에 쉽게 확인**할 수 있는 단일 HTML 페이지 대시보드 구축.

- 백엔드 없이 순수 HTML/CSS/JS로 동작
- 브라우저에서 바로 열어 사용 (파일 기반)
- 국내 기관(식약처·복지부·심평원)과 해외 기관(FDA·EMA·WHO) 기사를 통합 제공
- 사용자 상호작용(즐겨찾기, 메모, 읽음 처리 등)은 localStorage에 저장

## 4. 기능 명세

### 4-1. 탭 필터
- **전체 / 시장 전망 / 의약품 출시 / 정책·규제** (국내 통합)
- **FDA / EMA / WHO** (해외)
- 각 탭에 기사 수 표시, 미읽음 기사가 있으면 빨간 점(●) 표시

### 4-2. 날짜 범위 필터
- 전체 / 최근 6개월 / 3개월 / 1개월

### 4-3. 그룹 토글
- **연도별**: 2026년 / 2025년 섹션 구분
- **출처별**: 언론사·기관별 그룹
- **이슈별**: 9개 이슈 클러스터로 분류 (아래 4-7 참조)

### 4-4. 검색
- 기사 제목(국문+영문) 및 출처 대상 실시간 검색
- 검색어 하이라이트 (`<mark>` 태그)
- 검색어 초기화(✕) 버튼

### 4-5. 카드 기능
| 기능 | 동작 |
|---|---|
| 읽음 처리 | 기사 클릭 시 자동 처리, 카드 반투명 처리 |
| 즐겨찾기 ★ | localStorage 저장, 즐겨찾기 섹션에 상시 노출 |
| 메모 ✏️ | 카드별 메모 입력/저장/삭제, localStorage 저장 |
| 클립보드 복사 📋 | 기사 제목 + URL을 클립보드에 복사, 토스트 알림 |
| NEW 뱃지 | `isNew:true` 기사에 빨간 NEW 뱃지 + pulse 애니메이션 |
| Featured 카드 | 각 그룹의 첫 번째 기사가 "★ 주목 기사" 강조 표시 (전체 너비) |
| 교차 이슈 태그 | 이슈별 보기에서 다른 클러스터에도 해당되는 기사에 `↗ 태그` 표시 |

### 4-6. 키워드 트렌드 시각화
- 기사 제목 기준 10개 키워드 등장 빈도 수평 바 차트
- 기사 수 순 정렬, 0건 키워드 자동 숨김

### 4-7. 이슈별 그룹 (9개 클러스터)
| 이슈 | 주요 키워드 |
|---|---|
| GLP-1 · 비만치료제 | GLP-1, 비만치료, Wegovy |
| ADC 항암제 | ADC, 항체약물, Enhertu |
| CDMO · 위탁개발 | CDMO, 삼성바이오, 바이오로직스 |
| AI · 디지털 신약 | AI, 인공지능, 디지털치료 |
| 건강보험 · 약가 | 건강보험, 약가, 급여, 건정심 |
| 관세 · 글로벌 규제 | 관세, 트럼프, tariff |
| K-바이오 · 시장 | K-바이오, 바이오시밀러 |
| FDA 승인 동향 | FDA, 승인, approval |
| EMA · WHO 정책 | EMA, WHO, 팬데믹 |

- **중복 방지**: 각 기사는 첫 번째 매칭 그룹에만 배정
- **교차 태그**: 2개 이상 그룹에 해당하는 기사에 보조 이슈 태그 표시

### 4-8. 정책 변화 섹션 (상시 표시)
- 식약처·복지부·심평원 최신 5건 리스트 형태 표시
- 접기/펼치기 버튼, 상태 세션 유지

### 4-9. 해외 기관 섹션 (상시 표시)
- FDA·EMA·WHO 최신 6건 리스트 형태 표시
- 국문 제목 + 원문 영어 제목 함께 표시
- 접기/펼치기 버튼, 상태 세션 유지

### 4-10. 필터 상태바
- 툴바 아래에 상시 표시
- 현재 활성 필터(키워드 칩, 기간 칩, 검색어 칩) 표시
- 필터링된 기사 수 및 미읽음 기사 수 표시

### 4-11. 스크롤 위치 복원
- 페이지 스크롤 위치를 sessionStorage에 지속 저장
- 새로고침 시 이전 스크롤 위치로 자동 복원

### 4-12. 섹션 접기/펼치기
- 정책 변화 섹션, 해외 기관 섹션에 적용
- 접힘 상태는 sessionStorage에 저장하여 새로고침 후에도 유지
- 접힘 시 화살표 아이콘 회전 애니메이션

## 7. 자동 업데이트 시스템

### 7-1. 업데이트 주기
- **매일 오전 8시 (KST)** GitHub Actions가 자동 실행
- `workflow_dispatch`로 GitHub Actions 탭에서 수동 즉시 실행 가능

### 7-2. 파일 구조
```
pharma-news/
├── news.html                          # 메인 페이지
├── scripts/
│   ├── update_news.py                 # RSS 수집·업데이트 스크립트
│   └── requirements.txt               # Python 의존성
└── .github/
    └── workflows/
        └── daily_update.yml           # GitHub Actions 워크플로우
```

### 7-3. 자동화 흐름
```
매일 08:00 KST
      │
      ▼ GitHub Actions (ubuntu-latest, Python 3.11)
scripts/update_news.py 실행
      │
      ├─ 1. RSS 수집 (국내 6개 + 해외 3개 피드)
      ├─ 2. 관련성 키워드 필터링
      ├─ 3. kw 자동 분류 (제목 기반)
      ├─ 4. 해외 기사 영문 → 국문 번역 (Google Translate)
      ├─ 5. 중복 제거 (URL + ID 해시 기반)
      ├─ 6. isNew 만료 처리 (14일 이상 → isNew 제거)
      └─ 7. news.html ARTICLES 배열 상단 삽입 → 커밋·푸시
```

### 7-4. RSS 수집 소스
| 분류 | 소스 | kw |
|---|---|---|
| 국내 | 식품의약품안전처 | mfds |
| 국내 | 메디게이트뉴스 | auto |
| 국내 | 팜뉴스 | auto |
| 국내 | 히트뉴스 | auto |
| 국내 | 약사공론 | auto |
| 국내 | 바이오타임즈 | auto |
| 해외 | FDA (신약 승인 + 보도자료) | fda |
| 해외 | EMA | ema |
| 해외 | WHO | who |

- `kw: 'auto'` 소스는 기사 제목을 분석해 mfds/mohw/hira/launch/market 중 자동 분류

### 7-5. 자동 생성 기사 ID 규칙
- 형식: `r` + URL MD5 해시 앞 7자리 (예: `r3f8a2c1`)
- 수동 입력 기사: `a01`, `b01` 등 알파벳+숫자 조합
- 중복 방지: 동일 URL은 항상 동일 ID → 재수집해도 중복 삽입 없음

### 7-6. 수동 보완 방법
1. **중요 기사 수동 추가**: Claude Code에 "이 기사 추가해줘 + URL" 요청
2. **잘못된 기사 제거**: news.html ARTICLES 배열에서 해당 항목 삭제
3. **번역 품질 수정**: `titleKo` 필드 직접 수정
4. **즉시 실행**: GitHub → Actions 탭 → `제약시장 뉴스 자동 업데이트` → Run workflow

### 7-7. Python 의존성
| 패키지 | 용도 |
|---|---|
| `feedparser` | RSS/Atom 피드 파싱 |
| `deep-translator` | 영문 → 국문 번역 |
| `requests` | HTTP 요청 |
| `python-dateutil` | 날짜 파싱 |

## 8. UI 레이아웃 상세

### 8-1. 2컬럼 레이아웃 구조

```
┌─────────────────────────────────────────────────────┐
│  MASTHEAD                                            │
├─────────────────────────────────────────────────────┤
│  TOOLBAR (sticky)                                    │
├─────────────────────────────────────────────────────┤
│  FILTER BAR                                          │
├───────────────────────────┬─────────────────────────┤
│  layout-main (flex:1)     │  layout-side (296px)     │
│                           │  position:sticky         │
│  ★ 즐겨찾기               │  top:94px                │
│                           │  align-self:start        │
│  메인 기사 (카드/리스트)   │                          │
│                           │  📈 키워드 트렌드        │
│                           │  ┌──────────┬──────────┐ │
│                           │  │📋 정책변화│🌐 해외동향│ │
│                           │  └──────────┴──────────┘ │
└───────────────────────────┴─────────────────────────┘
```

- `max-width: 1200px`, 컬럼 간격 22px
- 900px 이하: 단일 컬럼으로 자동 전환, 사이드바 `position:static`
- 560px 이하: 정책·해외 섹션도 세로 배치로 전환

### 8-2. 리스트/카드 뷰 토글

| 모드 | 설명 |
|---|---|
| ▦ 카드 뷰 (기본) | `grid-template-columns: repeat(auto-fill, minmax(285px, 1fr))` 그리드 |
| ☰ 리스트 뷰 | 한 줄 compact 행, 같은 화면에 2~3배 많은 기사 표시 |

리스트 뷰에서 숨겨지는 요소: 영문 제목, 메모 미리보기, 메모 입력창, 교차 이슈 태그, Featured 레이블

### 8-3. 사이드바 sticky 동작 규칙

- `position: sticky; top: 94px; align-self: start`
- `top: 94px` = 툴바(~52px) + 필터바(~38px) + 여백
- `align-self: start` 필수 — 없으면 그리드 셀 높이에 맞춰 늘어나 sticky 동작 불가
- `max-height` + `overflow-y: auto` 금지 — 사이드바 하단 잘림 유발

## 10. 검증 가설

### 10-1. 자동화 시스템

| 가설 | 검증 방법 | 실패 기준 |
|---|---|---|
| RSS 피드 9개 모두 정상 동작할 것이다 | Actions 로그에서 `[WARN]` 빈도 확인 | 3개 이상 지속 실패 |
| Google 번역 품질이 업무에 충분할 것이다 | 1주일치 번역 기사 직접 검토 | 제목 의미 왜곡 30% 이상 |
| 키워드 필터로 관련 없는 기사가 걸러질 것이다 | 수집된 기사 중 무관 기사 비율 측정 | 무관 기사 20% 초과 |
| `kw: auto` 분류가 정확할 것이다 | 자동 분류된 기사 샘플 100건 수동 검토 | 오분류 25% 초과 |
| 중복 제거 로직이 동일 기사를 막을 것이다 | 같은 기사가 여러 번 나타나는지 관찰 | 중복 기사 발견 |
| 하루에 실질적인 신규 기사가 존재할 것이다 | 1개월 평균 일별 추가 건수 측정 | 평균 0건인 날이 절반 초과 |

### 10-2. 사용자 경험(UX)

| 가설 | 검증 방법 |
|---|---|
| 연도별 보기가 가장 많이 쓰일 것이다 | 실제로 어느 그룹 탭을 주로 쓰는지 1주 후 체감 |
| Featured 카드가 클릭률이 높을 것이다 | 즐겨찾기 기사 중 Featured였던 비율 확인 |
| 메모 기능을 실제로 사용할 것이다 | 2주 후 메모 저장된 기사 수 확인 |
| 정책 변화·해외 섹션을 매일 열어볼 것이다 | 접힘 상태 sessionStorage 값 관찰 |
| 검색보다 탭 필터를 더 많이 쓸 것이다 | 체감 사용 패턴 |

### 10-3. 데이터·콘텐츠 품질

| 가설 | 검증 방법 |
|---|---|
| 하루 수집 기사 중 실제 읽을 만한 기사가 있을 것이다 | 1주 후 즐겨찾기·읽음 처리 비율 |
| 이슈 9개 클러스터가 실제 관심 토픽을 커버할 것이다 | "이슈에 해당 없는" 중요 기사 발생 빈도 |
| 14일 isNew 기준이 적절할 것이다 | NEW 뱃지가 너무 많거나 너무 빨리 사라지는지 |
| 국내+해외 기사 비율(현재 약 3:2)이 적절할 것이다 | 해외 기사를 실제로 클릭하는지 |

### 10-4. 수집 제외 규칙

| 규칙 | 대상 예시 | 비고 |
|---|---|---|
| 명시적 식품 키워드 포함 시 즉시 제외 | 식품안전, 건강기능식품, 식품첨가물, 식품위생, 식품표시, 식품제조 | 의약품 키워드 여부 무관 |
| `'식품'` 포함 + 의약품 키워드 없음 → 제외 | "식품안전나라 개편" | 식약처 RSS의 식품 행정 기사 차단 |
| `'식품'` 포함 + 의약품 키워드 함께 있음 → 수집 | "식약처, 의약품·식품 통합 관리 강화" | 의약품 관련이면 허용 |

의약품 판별 기준 키워드: `의약품`, `의약`, `신약`, `제약`, `치료제`, `임상`, `바이오`

### 10-5. 우선 검증 순서

1. **자동화가 실제로 돌아가는가** — Actions 첫 실행(익일 08:00) 결과 확인
2. **수집된 기사가 관련 있는가** — 첫 1주일치 기사 품질 직접 확인
3. **실제로 매일 열어보게 되는가** — 2주 후 습관화 여부

> 2번·3번 결과에 따라 RSS 소스 교체, 키워드 필터 조정, UI 개편 방향 결정
